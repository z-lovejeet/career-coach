🎯 1. Problem Statement

Engineering students in India face a fragmented, overwhelming, and generic placement preparation experience:

  • No personalized guidance — Students use the same YouTube playlists and roadmaps regardless of their current skill level, goals, or target companies.
  • No self-awareness — Students don't know their actual strengths, weaknesses, or placement readiness.
  • Disconnected tools — Resume analysis, mock interviews, task planning, and job search happen on separate platforms with no integration.
  • No real-time job data — Students prepare blindly without knowing what companies are actually hiring right now.
  • Inconsistent effort — Without accountability (streaks, tracking, gamification), motivation drops within weeks.
⚠️ The Core Gap
There is no single platform that combines AI-powered profile analysis, personalized task planning, real-time job matching, mock interviews with feedback, and career mentorship — all adapting to the student's progress in real time.

💡 2. Our Approach & Solution

CareerAI is a multi-agent AI platform where 6 specialized AI agents collaborate to provide an end-to-end career coaching experience. Each agent has a focused responsibility, a typed input/output schema, and communicates through a shared database.

Key Design Principles

  1. Agent Specialization — Each AI agent has one job and does it expertly, rather than a single monolithic prompt doing everything.
  2. Real Data Pipeline — Profile → Analysis → Recommendations → Tasks → Interviews. Each agent's output feeds into the next.
  3. Hybrid Intelligence — AI-generated company matches + real-time LinkedIn job data from Apify.
  4. Honest AI — We instruct Gemini to be brutally honest about readiness, not sugarcoat assessments.
  5. Gamification — Streak tracking, XP, levels, achievements — all computed from real database activity (zero mock data).

🚫 5. Why Not LangChain?

This is a common question from judges. Here's our deliberate technical reasoning:

❌ LangChain Problems

  • Massive bundle — adds 2-5MB+ to server, significant cold start penalty on serverless (Vercel)
  • Abstraction overhead — chains, agents, tools, memory managers — we don't need 90% of it
  • Opaque debugging — when a chain fails, you're debugging 5 layers of abstraction
  • Version instability — frequent breaking API changes
  • Vendor lock-in — LangChain's abstractions make it harder to switch models

✅ Our Approach Instead

  • Direct API calls — @google/genai SDK, ~50KB, zero cold start
  • Single shared function — callGemini() — all 6 agents call through one 40-line function
  • TypeScript-typed I/O — each agent has a defined Input/Output type
  • JSON mode — Gemini's native responseMimeType: 'application/json', no parsing hacks
  • Agent = function — each agent is just an exported async function with a system prompt

The Core: callGemini() — Our "Framework"

Instead of LangChain's 100+ classes, we have one 40-line function that every agent calls through:

// src/lib/agents/gemini.ts — The ENTIRE "framework"
export async function callGemini<T>({
  systemPrompt,     // Agent's role & rules
  userPrompt,       // Context + question
  jsonMode = true,  // Gemini native JSON output
  temperature = 0.7,
  maxTokens = 8192,
}: GeminiCallOptions): Promise<T> {
  const response = await ai.models.generateContent({
    model: 'gemini-2.5-flash',
    contents: userPrompt,
    config: {
      temperature,
      maxOutputTokens: maxTokens,
      responseMimeType: jsonMode ? 'application/json' : undefined,
      systemInstruction: systemPrompt,
    },
  });
  return jsonMode ? JSON.parse(response.text) : response.text;
}
✅ Result
6 specialized agents, zero external framework dependencies. Total agent layer code: ~500 lines. A LangChain equivalent would be 2000+ lines with 15+ imported classes.

🧠
Agent 1: Profile Analyzer
src/lib/agents/analyzer.ts → /api/analyze

Purpose: Analyzes a student's complete profile (skills, resume, projects, education) and produces a placement readiness assessment.

Temperature: 0.4 (low — we want consistent, factual analysis, not creative output)

📥 Input
  • Skills list + self-ratings (1-10)
  • Projects (name, tech stack, description)
  • Education level & field
  • Experience level (fresher/intern/1-3yr)
  • Career goals
  • Resume text (extracted from PDF)
📤 Output
  • extracted_skills[] — name, level, confidence
  • strengths[] — skill + evidence
  • weaknesses[] — skill + reason
  • missing_skills[] — skill + importance
  • readiness_score — 0-100
  • summary — 2-3 sentence assessment

Storage: Output is saved to the analyses table in Supabase for historical tracking.

🏢
Agent 2: Company Recommender
src/lib/agents/recommendation.ts → /api/companies

Purpose: Matches the student with REAL Indian tech companies based on their analysis results. Split into "Current Fit" (hire-ready now) and "Target" (aspirational).

Temperature: 0.5 (balanced — needs to know real companies but also personalize)

📥 Input (from Analyzer output)
  • Readiness score
  • Extracted skills + strengths + weaknesses
  • Missing skills + importance
  • Career goals + experience level
  • Preferred locations
📤 Output
  • current_fit_companies[] — name, role, match%, skills
  • target_companies[] — name, gap skills, timeline
  • skill_gaps[] — current vs required level
  • roadmap_to_target[] — month-by-month plan

Key Constraint: System prompt mandates ONLY real Indian-market companies (Google, Flipkart, TCS, Razorpay, etc.) — no fictional companies.

📋
Agent 3: Task Planner
src/lib/agents/planner.ts → /api/tasks/generate

Purpose: Generates a personalized weekly task plan based on the analysis + company recommendations. Tasks reference real platforms (LeetCode, GFG) and real problem names.

Temperature: 0.6 (creative enough for varied tasks, structured enough for actionable plans)

📥 Input (from Analyzer + Recommender)
  • Analysis results (score, skills, weaknesses)
  • Recommendation results (target companies, gaps)
  • Current week number
  • Available hours/day
  • Career goals
📤 Output
  • weekly_plan.theme — week's focus area
  • tasks[] — title, description, category
  • Each task: difficulty, type (daily/weekly/milestone)
  • Day assignment (1-7), priority, estimated minutes
  • Categories: dsa, web_dev, system_design, soft_skills

Storage: Tasks saved to tasks table. Completion tracked with timestamps for streak calculation.

🎤
Agent 4: Interview Conductor
src/lib/agents/interview.ts → /api/interview

Purpose: Two sub-functions: (1) Generate interview questions by topic + difficulty, and (2) Evaluate student answers with scoring and feedback.

Temperatures: Generation = 0.6 (creative questions), Evaluation = 0.3 (consistent scoring)

📥 Generate Input
  • Topic (DSA, System Design, etc.)
  • Difficulty (easy/medium/hard)
  • Student level
  • Number of questions
📤 Evaluate Output
  • score — 0-10 per question
  • feedback — detailed review
  • topics_covered[] + topics_missed[]
  • improvement_tip — actionable advice

Storage: Questions, answers, and evaluations stored in interviews table. Overall score computed and saved.

💬
Agent 5: AI Mentor (Chat)
src/lib/agents/chat.ts → /api/chat

Purpose: Conversational AI mentor with personality. Knows the student's skills, goals, and readiness score. Provides career advice, doubt solving, and encouragement.

Temperature: 0.7 (conversational, friendly, but accurate)

📥 Input
  • Student message
  • Last 10 chat messages (context window)
  • Student context: skills, readiness score, goals
📤 Output
  • response — markdown-formatted reply
  • suggested_actions[] — follow-up buttons

Storage: Full chat history in chat_messages table. Loaded on page open, persists across sessions.

🗺️
Agent 6: Roadmap Architect
src/app/api/roadmap/route.ts → /api/roadmap

Purpose: Generates a brutally honest, week-by-week career preparation roadmap based on the student's dream company, timeline, and current skills. Includes feasibility assessment and skill gap analysis.

Temperature: 0.6 · Max tokens: 16,000 (large output for detailed weekly plans)

📥 Input
  • Dream company + target role
  • Timeline (1-12 months)
  • Available hours/day
  • Focus areas selected by user
  • Full profile + latest AI analysis + project count
📤 Output
  • overview — feasibility, readiness projection
  • phases[] — each with weeks, each week with tasks
  • keySkillsToAcquire[] — priority-ranked gaps
  • interviewPrep — DSA target, system design topics
  • warnings[] — honest challenges

Key Feature: If the dream is unrealistic in the timeframe, the AI says so and suggests alternative companies.

🔒 11. Security & Authentication

Authentication

  • Provider: Supabase Auth (Google OAuth)
  • Session: HTTP-only cookies via @supabase/ssr
  • Middleware: Next.js middleware protects all /dashboard, /profile, /roadmap, etc. routes
  • API Auth: Every API route calls supabase.auth.getUser() and returns 401 if missing

Data Security

  • RLS: Row-Level Security on ALL 8 tables
  • Policy: auth.uid() = profile_id — users see only their own data
  • API Keys: GEMINI_API_KEY and APIFY_API_TOKEN are server-side only (never exposed to client)
  • Resume Storage: Uploaded to Supabase Storage bucket with auth policies
